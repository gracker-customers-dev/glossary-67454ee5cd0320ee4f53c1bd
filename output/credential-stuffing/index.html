Credential stuffing

A cyber attack method where stolen account credentials are used to gain unauthorized access to user accounts on various platforms.

Description

Credential stuffing is a type of cyber attack that exploits the tendency of users to reuse the same login credentials across multiple websites and applications. Attackers obtain sets of usernames and passwords, often from previous data breaches, and then use automated tools to test these credentials on a wide range of online services. Because many users do not create unique passwords for different accounts, a successful login can occur even if the credentials were originally compromised from just one site. This method poses a significant threat to non-human identities, such as bots or automated accounts, which can be generated to mimic legitimate users. These non-human entities can exploit access to automate fraudulent activities, such as making purchases, sending spam, or scraping data. Credential stuffing attacks can lead to significant financial losses for organizations and compromise personal data security for users. To mitigate these risks, companies are encouraged to implement measures such as multi-factor authentication and monitoring for unusual login patterns.

Examples

  • An attacker uses credentials leaked from a data breach to access multiple user accounts on an e-commerce site.
  • A bot utilizes credential stuffing to gain access to social media accounts, allowing it to spread misinformation or spam.

Additional Information

  • Credential stuffing is often coupled with automated tools that can rapidly try thousands of username-password combinations.
  • Organizations can protect themselves by encouraging users to adopt unique passwords and enabling multi-factor authentication.
