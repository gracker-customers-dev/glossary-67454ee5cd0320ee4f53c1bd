Two-factor authentication

A security process requiring two different forms of identification to verify a non-human identity.

Description

Two-factor authentication (2FA) is a security mechanism that enhances the protection of non-human identities, such as automated systems, bots, and other digital entities. In the context of non-human identities, 2FA typically involves the use of two distinct methods of verification before granting access to sensitive resources or systems. The first factor is usually something the non-human entity knows, like an API key or password, while the second factor is often something the entity possesses, such as a cryptographic token, a certificate, or a response generated by an authentication app. This layered approach significantly reduces the risk of unauthorized access, as an attacker would need both factors to compromise the non-human identity. 2FA is particularly relevant in environments where automated processes interact with sensitive data or perform critical functions, ensuring that only legitimate automated identities can execute operations. By implementing 2FA, organizations can bolster their security posture against potential threats and breaches.

Examples

  • A bot accessing an API must provide both a password and a time-based one-time password (TOTP).
  • A cloud service uses a certificate along with a secret key to authenticate automated backup scripts.

Additional Information

  • 2FA can help mitigate the risks of credential theft and unauthorized access.
  • Implementing 2FA for non-human identities is becoming a best practice in cybersecurity.
