Second Factor Authentication

A security process that requires two different forms of identification to verify a user's identity.

Description

Second Factor Authentication (2FA) enhances security by requiring not just a password (the first factor) but also a second form of verification. This is especially crucial for non-human identities, such as application accounts, APIs, or automated processes, where traditional username/password combinations may not suffice due to the risk of compromise. Non-human identities often interact with systems in ways that can be vulnerable to unauthorized access. By implementing 2FA, organizations can ensure that even if a password is stolen, an attacker would still need the second factor, which is typically something only the legitimate user has access to, such as a mobile device, hardware token, or biometric verification. This greatly reduces the risk of unauthorized actions being taken by automated systems or malicious actors pretending to be legitimate services. 2FA is a critical component in modern cybersecurity strategies, helping to protect sensitive data and operations by adding an additional layer of security beyond the password.

Examples

  • Using a mobile app like Google Authenticator to generate time-based one-time passwords (TOTPs) for an API access.
  • Receiving a text message with a verification code after submitting a password for a non-human service account.

Additional Information

  • 2FA can use various methods, including SMS, email, phone calls, or authentication apps.
  • Many organizations are adopting 2FA as a requirement for accessing sensitive systems or data.
